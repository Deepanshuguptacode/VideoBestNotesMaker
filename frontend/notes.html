ğŸš€ Part 1: Weight Initialization - Setting the Stage for Success

ğŸ’¡ The Problem with Bad Initializations

  • ğŸ“Œ All Zeros: All neurons learn the same thing, breaking symmetry.
  • ğŸ“Œ Small Random Values: Can lead to vanishing gradients (especially in deep networks). Think of it like a whisper that fades away as it travels down a long hallway.
  • ğŸ“Œ Large Random Values: Can lead to exploding gradients. Like shouting so loud that it overpowers everything.

ğŸ” The Solution: Smart Initialization Strategies

We need to initialize weights in a way that keeps the variance of activations and gradients reasonable throughout the network.

ğŸ“ 1. Xavier/Glorot Initialization

  • ğŸ“Œ Definition: A weight initialization technique designed to keep the variance of activations and gradients consistent across layers.
  • ğŸ“Œ Suitable For: Layers with sigmoid or tanh activation functions.
  • ğŸ“Œ Why It Works: Sigmoid and tanh have a limited range (-1 to 1). Xavier helps prevent saturation.
  • ğŸ“Œ Formula:
    • Uniform Distribution: Weights are sampled from a uniform distribution between [-limit, limit], where limit = sqrt(6 / (fan_in + fan_out))
    • Normal Distribution: Weights are sampled from a normal distribution with mean 0 and standard deviation sqrt(2 / (fan_in + fan_out))
    • fan_in: Number of inputs to the layer
    • fan_out: Number of outputs from the layer
  • ğŸ¯ Analogy: Imagine tuning a radio. Xavier initialization tries to balance the signal strength coming in and going out so you get a clear sound.

ğŸ“ 2. He Initialization

  • ğŸ“Œ Definition: Specifically designed for layers using ReLU (Rectified Linear Unit) or its variants (Leaky ReLU, Parametric ReLU).
  • ğŸ“Œ Suitable For: Layers with ReLU activations.
  • ğŸ“Œ Why It Works: ReLU sets negative values to zero, which can reduce the variance. He initialization compensates for this.
  • ğŸ“Œ Formula:
    • Uniform Distribution: Weights are sampled from a uniform distribution between [-limit, limit], where limit = sqrt(6 / fan_in)
    • Normal Distribution: Weights are sampled from a normal distribution with mean 0 and standard deviation sqrt(2 / fan_in)
    • fan_in: Number of inputs to the layer
  • ğŸ¯ Analogy: Think of a garden hose that sometimes gets kinked. He initialization is like making the hose a bit wider so that even when it's kinked, enough water still flows through.
  • ğŸ“Œ Problem with ReLU activations: This issue is referred to as â€œdying ReLUâ€
  • ğŸ“Œ Choosing a Value * He Initialization -> sqrt(2/n)

ğŸš€ Code Example (Keras - He Initialization)

        
import numpy as np
import keras
from keras.models import Sequential
from keras.layers import Dense
from keras.initializers import he_normal
from keras.utils import to_categorical
import matplotlib.pyplot as plt

# 1. Data Preparation:  Generating Random Data
num_samples = 1000
input_dim = 784
num_classes = 10

# Generating random data for demonstration purposes
x_train = np.random.rand(num_samples, input_dim) #Input data with random values
y_train = to_categorical(np.random.randint(num_classes, size=(num_samples,))) # One-hot encode random labels

# 2. Building the Model with He Initialization
model_with_he = Sequential()
model_with_he.add(Dense(128, input_dim=input_dim, activation='relu', kernel_initializer=he_normal()))
model_with_he.add(Dense(64, activation='relu', kernel_initializer=he_normal()))
model_with_he.add(Dense(num_classes, activation='softmax', kernel_initializer=he_normal()))

# 3. Compiling the Model
model_with_he.compile(loss='categorical_crossentropy', optimizer='adam', metrics=['accuracy'])

# 4. Model Summary
model_with_he.summary()

# 5. Training the Model
history_he = model_with_he.fit(x_train, y_train, epochs=10, batch_size=32, validation_split=0.2, verbose=0)

# 6. Plotting the Loss
plt.plot(history_he.history['loss'], label='He Initialization')
plt.plot(history_he.history['val_loss'], label = 'Validation He Initialization')
plt.xlabel('Epoch')
plt.ylabel('Loss')
plt.title('Training Loss with He Initialization')
plt.legend()
plt.show()
  • ğŸ“Œ Explanation:
    • kernel_initializer=he_normal(): Specifies that the weights of the Dense layer should be initialized using the He initialization method with a normal distribution. he_uniform() is also available for the uniform distribution.
    • plt.plot(history_he.history['val_loss'], label = 'Validation He Initialization'): used to evaluate the training of your model and if it will work when real data is introduced.

ğŸ’¡ The Problem: Internal Covariate Shift

  • ğŸ“Œ Definition: The change in the distribution of network activations as the parameters change during training. Each layer is constantly trying to adapt to a moving target.
  • ğŸ¯ Analogy: Imagine trying to hit a target that's constantly moving randomly. It's much harder than hitting a stationary target.
  • ğŸ“Œ Internal Covariate Shift Definition : The distribution of data through different layers changes as the weights and biases at each layer change.
  • ğŸ“Œ Why it is Needed?: In a deep neural network, each layer initializes weights randomly or gets adjusted during training. These changes alter the distribution of data as it flows through the network. If there is covariant shift, it becomes harder for subsequent layers to learn.

ğŸ” The Solution: Normalizing Activations Within Mini-Batches

  • ğŸ“Œ Definition: A technique that normalizes the activations of a layer for each mini-batch during training.
  • ğŸ“Œ How It Works:
    1. Calculate Mean and Variance: Compute the mean (Î¼) and variance (ÏƒÂ²) of the activations for the current mini-batch.
    2. Normalize: Normalize the activations by subtracting the mean and dividing by the square root of the variance (adding a small epsilon to avoid division by zero). x_norm = (x - Î¼) / sqrt(ÏƒÂ² + Îµ)
    3. Scale and Shift: Apply a learnable scale (Î³) and shift (Î²) to the normalized activations. y = Î³ * x_norm + Î²
      • Î³ (gamma): Learnable scale parameter
      • Î² (beta): Learnable shift parameter
  • ğŸ¯ Analogy: Imagine standardizing exam scores in a class. Batch normalization is like doing that for the activations within each layer of your neural network.

ğŸ’¡ Benefits of Batch Normalization

  • ğŸ“Œ Faster Training: Allows for higher learning rates.
  • ğŸ“Œ Regularization: Reduces the need for dropout (sometimes).
  • ğŸ“Œ Reduces Internal Covariate Shift: Stabilizes learning.
  • ğŸ“Œ More Robust: Less sensitive to weight initialization.
  • ğŸ“Œ Noise Reduction: Reduces noise in the input data that can cause bad values with the gradient.
  • ğŸ“Œ Improved Gradient Flow: Helps in maintaining the gradients, especially in deep networks.

ğŸš€ Code Example (Keras - Batch Normalization)

        
import numpy as np
from keras.models import Sequential
from keras.layers import Dense, BatchNormalization, Activation
from keras.optimizers import Adam
from keras.utils import to_categorical

# 1. Data Generation (Example):
x_train = np.random.rand(1000, 784) #1000 samples, each with 784 features
y_train = to_categorical(np.random.randint(0, 10, size=(1000,)), num_classes=10) # One-hot encode random labels


# 2. Model Definition with Batch Normalization:
model = Sequential()
model.add(Dense(128, input_shape=(784,)))
model.add(BatchNormalization())
model.add(Activation('relu'))
model.add(Dense(64))
model.add(BatchNormalization())
model.add(Activation('relu'))
model.add(Dense(10, activation='softmax'))

# 3. Model Compilation:
model.compile(optimizer='Adam', loss='categorical_crossentropy', metrics=['accuracy'])

# 4. Model Summary
model.summary()

# 5. Training the Model:
model.fit(x_train, y_train, epochs=5, batch_size=32)
  • ğŸ“Œ Explanation:
    • BatchNormalization(): This layer normalizes the activations of the previous layer. It learns the Î³ (scale) and Î² (shift) parameters during training.
    • Placement: It's common to place BatchNormalization after a fully connected (Dense) or convolutional layer and before the activation function.
